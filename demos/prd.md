# Cross-Section Explorer — PRD v2

**Standard:** GM: G-GMD.B.4  
**Cluster:** Geometric Measurement and Dimension — Visualize relationships between 2D and 3D objects  
**Component of Rigor:** Conceptual Understanding  
**Replaces:** Two dead Shodor Flash links in the Louisiana Geometry Teacher's Companion  
**Repository:** `creative-lab-demos` (standalone Vercel deploy)  
**Portfolio role:** Act 1 Live Demo — the experience that stops an ISTE educator mid-stride  
**Stack:** React 19 + TypeScript + Vite + R3F + @react-three/csg + @react-three/rapier + @react-three/drei + GSAP + Tailwind CSS v4

---

## 1. Problem Statement

GM: G-GMD.B.4 requires students to:

- Identify 2D cross sections of 3D solids
- Identify 3D solids generated by rotating 2D shapes

The Louisiana Geometry Teacher's Companion lists two Shodor Flash applets as the recommended tools for both skills. Both links are dead. No functional replacement exists in the Louisiana standards ecosystem. The standard is taught with static diagrams and physical models where available.

This experience is the replacement. It is also the proof-of-concept for a 3D geometry component library that extends through G-GMD.A.1 (volume), G-GMD.A.3 (cylinder/cone/sphere formulas), and G-MG.A.1 (modeling with 3D shapes).

---

## 2. Learning Target

**Primary:** Students can identify the 2D shape produced when a plane cuts through a 3D solid at various angles and positions.

**Secondary:** Students can identify the 3D solid generated when a 2D silhouette is rotated about an axis.

**The insight that connects them:** The cross section of a solid of revolution is the silhouette that generated it. This is never stated — it is discovered when the student completes both modes with the same solid family.

**ALD anchor (G-GMD.B.4):**

- Level 2: Names the cross section of a familiar solid from a static diagram
- Level 3: Predicts the cross section before revealing it; names solids of revolution
- Level 4: Predicts non-obvious cross sections (hexagon from cube, ellipse from tilted cone)
- Level 5: Explains why the cross section has that shape using geometric reasoning

This experience targets Levels 3 and 4. Level 5 is a human conversation — not a module goal.

---

## 3. Audience

**Primary:** Portfolio visitors at ISTE 26 — educators, edtech builders, curriculum designers. They are evaluating whether this work is real, rigorous, and worth a conversation.

**Secondary:** Geometry teachers who find the standalone Vercel deploy via QR code or direct link. They need to be able to hand a phone to a student and have the experience be self-evident.

**Key constraint:** No instructions. If the interaction requires explanation before engagement, the experience has failed. Every affordance must be visually obvious.

## 4. Mode Architecture

The experience has two modes, not a linear stage flow. Modes are peer — neither is prerequisite to the other. The student chooses entry.

```
MODE A: CROSS SECTION
  Solid displayed in 3D space
  Cutting plane draggable along Y axis
  Plane tiltable on X axis
  Cross section face illuminates in amber as plane intersects solid
  Shape label appears after meaningful interaction

MODE B: ROTATION
  2D silhouette displayed against vertical axis
  Auto-rotate sweeps silhouette through 360°
  LatheGeometry builds solid in real time
  Solid materializes at completion
  Shape label appears after full rotation

CONNECTION MOMENT (earned, not scheduled)
  Appears only after student has completed both modes with the same solid
  A single sentence. Not a modal. Not an announcement.
```

**Deviation from Observe → Manipulate → Discover:** This demo has no Observe phase. The student lands inside Mode A — the cutting plane is already positioned mid-solid, the cross section is already visible. The first frame IS the experience. There is nothing to read before engaging.

**Justification:** Portfolio context demands immediate visceral impact. A setup screen loses the visitor. Geometry teachers who reach the demo via QR code are self-motivated — they don't need orientation. The experience is designed to be understood by doing.

## 5. The Four Solids

| Solid    | Cross sections available            | Rotation silhouette           |
| -------- | ----------------------------------- | ----------------------------- |
| Cone     | circle, ellipse, triangle, parabola | right triangle                |
| Cylinder | circle, ellipse, rectangle          | rectangle                     |
| Cube     | square, rectangle, hexagon          | — (not a solid of revolution) |
| Sphere   | circle (varies by position)         | semicircle                    |

**The hexagon is the centerpiece moment.** When the student tilts the cutting plane through a cube at ~45°, a regular hexagon emerges from the CSG intersection. This is the discovery that no static diagram communicates — the student has to see it happen. Every other cross section is expected. The hexagon is the proof that the experience is teaching something real.

**The cube has no Rotation Mode** — it is not a solid of revolution. The mode toggle is disabled for cube in Mode B. This is pedagogically correct and should be communicated without apology.

## 6. Architecture — CSG-First

### Why CSG changes everything

The original approach computed cross-section geometry manually — custom functions per solid, approximating ellipses and hexagons with vertex arrays. `@react-three/csg` performs real boolean operations on 3D meshes. The cross-section face is the actual intersection result, not an approximation.

```tsx
// The entire cutting interaction — no manual geometry math
import { Geometry, Base, Subtraction, useCSG } from "@react-three/csg";
import { PivotControls } from "@react-three/drei";

function CuttingGeometry({ solidGeometry }) {
  const csg = useRef();
  return (
    <mesh>
      <Geometry ref={csg} useGroups computeVertexNormals>
        <Base geometry={solidGeometry}>
          <meshStandardMaterial color={0x232018} transparent opacity={0.85} />
        </Base>
        <PivotControls onDrag={() => csg.current.update()}>
          <Subtraction>
            {/* Large box used as half-space cutter */}
            <boxGeometry args={[10, 10, 10]} />
            <meshStandardMaterial color={0xd4962a} />{" "}
            {/* amber — cross section face */}
          </Subtraction>
        </PivotControls>
      </Geometry>
    </mesh>
  );
}
```

`useGroups={true}` assigns the amber material to the exposed cross-section face automatically. `computeVertexNormals` ensures the cut face catches the amber point light correctly. `PivotControls` provides the drag handle with zero custom gesture code.

### Rapier physics mode

Physics is a deliberate flex — pedagogically optional, technically essential to demonstrate stack mastery.

```tsx
<Physics gravity={[0, -9.81, 0]}>
  <RigidBody
    ref={rigidBodyRef}
    type={physicsMode ? "dynamic" : "fixed"}
    colliders="hull"
    restitution={0.6}
    onContactForce={({ totalForce }) => {
      if (totalForce.y > 15) flashCrossSectionPlane();
    }}
  >
    <SolidMesh
      onClick={() =>
        rigidBodyRef.current?.applyImpulse(
          { x: (Math.random() - 0.5) * 4, y: 10, z: (Math.random() - 0.5) * 4 },
          true,
        )
      }
    />
  </RigidBody>
  <CuboidCollider position={[0, -3, 0]} args={[20, 0.5, 20]} />
</Physics>
```

`colliders="hull"` generates a convex hull automatically. `onContactForce` triggers a GSAP flash on the cross-section plane when the solid hits the floor — the physics and the geometry connect in a single moment.

### Rotation Mode — LatheGeometry

```tsx
function RotationScene({ solidId, rotationAngle }) {
  const geometry = useMemo(() => {
    const points = SILHOUETTE_POINTS[solidId];
    const segments = Math.max(3, Math.round((rotationAngle / 360) * 64));
    return new THREE.LatheGeometry(
      points,
      segments,
      0,
      (rotationAngle * Math.PI) / 180,
    );
  }, [solidId, Math.round(rotationAngle / 5)]); // bucket to every 5° to limit rebuilds

  return (
    <mesh geometry={geometry}>
      <meshStandardMaterial
        color={0x232018}
        transparent
        opacity={(rotationAngle / 360) * 0.85}
      />
    </mesh>
  );
}
```

The solid materializes as the rotation completes — opacity scales with angle. At 360°, GSAP animates to full opacity and the label appears.

### Performance contracts

| Concern          | Contract                                                               |
| ---------------- | ---------------------------------------------------------------------- |
| CSG recompute    | Only on `PivotControls` drag event — never in `useFrame`               |
| LatheGeometry    | Bucketed to 5° increments via `Math.round(angle / 5)` in `useMemo` key |
| Object reuse     | All `Vector3`, `Euler` instances in `useMemo` outside loops            |
| Rapier           | Lazy-loaded — zero cost until physics mode activated                   |
| Segment counts   | Halved on `pointer: coarse` devices                                    |
| Pixel ratio      | `dpr={Math.min(window.devicePixelRatio, 2)}`                           |
| Material sharing | One instance of each material shared across all renders                |

## 7. State Architecture

```ts
type SolidId = "cone" | "cylinder" | "cube" | "sphere";
type ModeId = "crossSection" | "rotation";

interface DemoState {
  mode: ModeId;
  solidId: SolidId;
  physicsMode: boolean;
  // Mode A state
  planeY: number; // [-1.5, 1.5] world units — controlled by PivotControls
  tiltAngle: number; // [-45, 45] degrees — controlled by PivotControls
  shapeLabel: string | null;
  labelVisible: boolean;
  // Mode B state
  rotationAngle: number; // [0, 360] degrees
  rotationComplete: boolean;
  // Connection moment
  completedModes: Set<`${SolidId}-${ModeId}`>;
  connectionVisible: boolean;
}
```

`useReducer` pattern. Actions: `SET_MODE`, `SET_SOLID`, `TOGGLE_PHYSICS`, `SET_PLANE_POSITION`, `REVEAL_LABEL`, `HIDE_LABEL`, `SET_ROTATION_ANGLE`, `COMPLETE_ROTATION`, `MARK_MODE_COMPLETE`, `SHOW_CONNECTION`.

**Connection moment trigger:**

```ts
case 'MARK_MODE_COMPLETE': {
  const next = new Set(state.completedModes).add(`${state.solidId}-${state.mode}`)
  const bothDone = next.has(`${state.solidId}-crossSection`) && next.has(`${state.solidId}-rotation`)
  return { ...state, completedModes: next, connectionVisible: bothDone && state.solidId !== 'cube' }
}
```

All copy follows the educational copywriter principle: direct attention, never explain the concept.

### Mode A labels (appear after meaningful drag interaction)

```ts
const CROSS_SECTION_LABELS = {
  // Cone
  "cone-top": "point",
  "cone-circle": "circle",
  "cone-ellipse": "ellipse",
  "cone-triangle": "triangle",
  "cone-parabola": "parabola",
  // Cylinder
  "cylinder-circle": "circle",
  "cylinder-ellipse": "ellipse",
  "cylinder-rect": "rectangle",
  // Cube
  "cube-square": "square",
  "cube-rect": "rectangle",
  "cube-hexagon": "hexagon", // ← the moment
  // Sphere
  "sphere-small": "circle",
  "sphere-great": "great circle",
};
```

Shape classification is derived from the CSG result geometry's bounding box and vertex count — not pre-calculated. This means unexpected cross sections (parabola from a steep cone tilt) are labeled correctly even if the student discovers them by accident.

### Mode B labels (appear after 360° rotation completes)

```ts
const ROTATION_LABELS = {
  cone: "cone",
  cylinder: "cylinder",
  sphere: "sphere",
};
```

### Connection moment copy

```
"That cross section — it's the shape you started with."
```

Single sentence. Fraunces italic. Amber. Centered below the canvas. Fades in over 0.6s. No dismiss button — it fades out automatically after 4s. It is a confirmation, not an announcement.

## 10. Build Order

Maintain a runnable, deployable state after every round.

**Round 1 — Scaffold + solids**
`npm create vite`, install stack. Four solids render with dark materials. OrbitControls. Mode bar and solid selector wired to state. Nothing interactive yet.

**Round 2 — CSG cutting plane**
`CuttingGeometry.tsx` with `<Geometry useGroups>` + `<Base>` + `<Subtraction>` + `PivotControls`.
**Gate:** Do not proceed until the hexagon appears from a diagonal cube cut. The hexagon is the validation that CSG is working correctly.

**Round 3 — Shape classification + labels**
`useShapeClassifier.ts` — derives label from CSG geometry bounding box and vertex count.
Natural position GSAP pulses on drag handle at apex, equator, and base positions.
All four solids fully functional in Mode A.

**Round 4 — Rotation Mode**
`silhouettes.ts` — Vector2 point arrays for cone, cylinder, sphere.
`useSolidRotation.ts` — LatheGeometry builder + GSAP auto-rotate timeline.
`RotationScene.tsx` — silhouette display + emerging solid.
Label reveal after 360° completion.

**Round 5 — Connection moment**
`MARK_MODE_COMPLETE` reducer logic.
`connectionVisible` state driving `ShapeLabel` connection copy.
Validate: complete both modes with cone → connection appears. Complete both with cube → connection does not appear (cube is not a solid of revolution).

**Round 6 — Rapier physics mode**
Lazy-load `Physics` + `RigidBody`. `colliders="hull"`. `applyImpulse` on click. `onContactForce` impact flash. Floor collider.

**Round 7 — Polish**
GSAP camera resets on solid change. Mobile segment reduction. Reduced-motion support (`prefers-reduced-motion` → disable auto-rotate, disable entrance animations). Performance audit.

**Round 8 — Export + deploy**
Export as lazy-loadable `CrossSectionExplorer` component.
Standalone Vercel deploy. QR code.
Integration into portfolio Live Demo section.

This experience is the seed of the `creative-lab-demos` 3D geometry library.

| Future standard                                    | Reuse from this module                         |
| -------------------------------------------------- | ---------------------------------------------- |
| G-GMD.A.1 — Volume via Cavalieri                   | `RotationScene` + `CuttingGeometry`            |
| G-GMD.A.3 — Cylinder/cone/sphere formulas          | All four solid geometries + materials          |
| G-MG.A.1 — Modeling with 3D shapes                 | `SolidMesh` as base component                  |
| 8.G.C.9 — Volume of cone/cylinder/sphere (Grade 8) | Entire module structure                        |
| M3 Pythagorean Theorem — 3D proof                  | `InstancedRigidBodies` (Rapier, found in scan) |

The architecture was designed for reuse from Round 1. `CuttingGeometry` accepts any `THREE.BufferGeometry`. `RotationScene` accepts any `Vector2[]` silhouette. These are not module-specific — they are geometry primitives.
